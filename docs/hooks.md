# Hooks

hive captures data from Claude Code via its hook system and from git via a post-commit hook.

## Claude Code Hooks

### Installation

`hive init` writes hook definitions into `.claude/settings.json` in the project directory:

```json
{
  "hooks": {
    "SessionStart": [
      {"type": "command", "command": "hive capture session-start"}
    ],
    "Stop": [
      {"type": "command", "command": "hive capture stop"}
    ],
    "PostToolUse": [
      {"type": "command", "command": "hive capture post-tool-use"}
    ],
    "PreCompact": [
      {"type": "command", "command": "hive capture pre-compact"}
    ]
  }
}
```

Hooks are additive -- if `.claude/settings.json` already has other hooks, hive appends its entries without duplicating.

### How Hooks Execute

Claude Code invokes each hook command and passes a JSON payload on **stdin**. The `hive capture <event>` CLI command:

1. Maps kebab-case CLI names to PascalCase event names (`session-start` -> `SessionStart`)
2. Reads JSON from stdin
3. Creates a `ClaudeCodeAdapter` and calls `adapter.handle(event_name, data)`

### Hook Events

#### SessionStart

**Fires**: When a new Claude Code session begins.

**Stdin payload**: `{"session_id": "...", "project_path": "/..."}`

**What it does**:
- Creates a session row in the database via `upsert_session()`
- Collects current git state (branch, commit SHA, remote URL, author) from the project directory
- Stores git metadata as enrichments (`source = "git"`, keys: `branch`, `commit`, `remote`)

#### Stop

**Fires**: When a Claude Code session ends.

**Stdin payload**: `{"session_id": "...", "project_path": "/...", "transcript_path": "/..."}`

**What it does**:
1. Resolves the transcript JSONL file (explicit `transcript_path` or convention-based lookup under `watch_path`)
2. Parses all lines, filtering to `type = "user" | "assistant"` and skipping `isMeta` entries
3. Normalizes roles (`user` -> `human`), extracts text from content blocks
4. Scrubs all content for secrets using configured patterns
5. Inserts messages into the database
6. Builds FTS index from scrubbed message content
7. Derives a summary from the first meaningful human message (skips slash commands, XML tags, tool results)
8. Updates the session row with `ended_at`, `message_count`, and `summary`
9. Runs all enrichers (Git, Files, Quality)
10. Auto-pushes to team server if sharing is enabled (see below)

#### PostToolUse

**Fires**: After Claude Code executes a tool call.

**Stdin payload**: `{"session_id": "...", "tool_name": "Read", "tool_input": {"file_path": "/..."}}`

**What it does**:
- Only processes tool calls to: `Read`, `Write`, `Edit`, `Bash`, `Glob`, `Grep`
- Extracts file paths from `tool_input` (structured keys for Read/Write/Edit/Glob/Grep, heuristic absolute-path extraction for Bash)
- Creates `session -> file` edges with relationship `"touched"`
- Duplicate edges are silently skipped

#### PreCompact

**Fires**: Before Claude Code compacts (summarizes) the conversation context.

**Stdin payload**: `{"session_id": "...", "transcript_path": "/..."}`

**What it does**:
- Reads the full transcript file
- Scrubs secrets from the raw text
- Stores as an enrichment (`source = "compact_snapshot"`, `key = "transcript"`)
- This preserves the full conversation before compaction truncates it

## Auto-Push Flow (Stop Hook)

When the `Stop` handler finishes processing, it calls `_maybe_push()`:

1. Load the per-project config from `<project>/.hive/config.toml`
2. If `sharing` is not enabled, return immediately
3. Export the full session payload (session + messages + enrichments + annotations + edges)
4. Scrub all string values in the payload with `scrub_payload()`
5. POST to `{server_url}/api/sessions` in a **daemon thread** so the hook returns without blocking Claude Code
6. Failures are logged at debug level and silently dropped

## Git Post-Commit Hook

**Source**: `src/hive/capture/git_hook.py`

### Installation

`hive init` installs a post-commit hook at `.git/hooks/post-commit`:

```bash
#!/usr/bin/env bash
# Generated by hive -- do not edit manually.
# Links git commits to the most recently active AI session.

if command -v hive &>/dev/null; then
    hive hook post-commit &
fi
```

If a post-commit hook already exists, hive appends its invocation to the end (preserving the existing hook). If hive is already present, installation is skipped.

### What It Does

1. Reads the HEAD commit SHA and changed file list from git
2. Finds the most recently ended session within the `link_window_minutes` window (default 30 minutes)
3. Creates a `session -> commit` edge with relationship `"produced"`
4. Creates `session -> file` edges with relationship `"committed"` for each file changed in the commit

This enables lineage queries that connect sessions to the commits they produced and the files those commits changed.
